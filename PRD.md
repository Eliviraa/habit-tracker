# PRD / Техническое задание: Трекер привычек

Версия: 1.0  
Статус: к реализации  
Технология: C++/Qt  
Язык: русский

---

## 1. Цель проекта

Создать настольное приложение, в котором пользователь может вести список привычек, ежедневно отмечать их выполнение и видеть свой прогресс.

---

## 2. Что входит в приложение (MVP)

- Создание привычки (название, описание)
- Редактирование привычки
- Удаление привычки
- Ежедневная отметка выполнения (да/нет)
- Отображение текущей серии выполнения (сколько дней подряд)
- Отображение лучшей серии
- График выполнения за последние 30 дней
- Хранение всех данных в базе SQLite

---

## 3. Что НЕ входит в первую версию

- Напоминания и уведомления
- Синхронизация с облаком
- Многопользовательский режим
- Экспорт/импорт данных

---

## 4. Пользовательские сценарии

### Сценарий 1: Создание привычки
1. Пользователь открывает приложение
2. Нажимает кнопку «Добавить привычку»
3. Вводит название «Пить воду»
4. Нажимает «Сохранить»
5. Привычка появляется в списке

### Сценарий 2: Отметка выполнения
1. Пользователь открывает приложение
2. Видит список привычек
3. Напротив привычки «Пить воду» нажимает чекбокс
4. Приложение сохраняет отметку за сегодня
5. Серия выполнения увеличивается на 1

### Сценарий 3: Просмотр прогресса
1. Пользователь нажимает на привычку в списке
2. Открывается окно со статистикой
3. Показывается: «Текущая серия: 5 дней», «Лучшая серия: 12 дней»
4. Показывается график выполнения за последние 30 дней

---

## 5. Модель данных

### Таблица «Привычки» (habits)

| Поле | Тип | Что хранит |
|------|-----|-----------|
| id | целое число | Уникальный номер привычки |
| name | текст | Название привычки |
| description | текст | Описание (можно оставить пустым) |
| created_at | дата и время | Когда создали привычку |

### Таблица «Отметки» (habit_logs)

| Поле | Тип | Что хранит |
|------|-----|-----------|
| id | целое число | Уникальный номер отметки |
| habit_id | целое число | К какой привычке относится отметка |
| date | дата | За какой день отметка |
| completed | число (0 или 1) | 1 = выполнено, 0 = не выполнено |

**Как это работает:**
- В таблице habits хранится список привычек
- В таблице habit_logs хранятся отметки за каждый день
- Одна привычка может иметь много отметок

---

## 6. Экранный интерфейс

### Главное окно
- Список всех привычек
- Кнопка «Добавить привычку»
- У каждой привычки: название, чекбокс для отметки, кнопка «Статистика»

### Окно добавления привычки
- Поле для названия (обязательное)
- Поле для описания (необязательное)
- Кнопки «Сохранить» и «Отмена»

### Окно статистики
- Текст: «Текущая серия: X дней»
- Текст: «Лучшая серия: Y дней»
- График выполнения за последние 30 дней

## 9. Критерии приемки

Приложение считается готовым, если:

1. Можно создать привычку и она появляется в списке
2. Можно отметить привычку как выполненную
3. После перезапуска программы все данные сохраняются
4. Серия выполнения считается правильно
5. График показывает отметки за 30 дней
